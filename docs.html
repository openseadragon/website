Quickstart

A working deep-zoom viewer in five minutes. Drop in a script tag, add a container, point at a tile source — that's the whole shape.

1. Install

Use the package manager you already use. OpenSeadragon ships as an ESM module, a UMD bundle, and a single-file IIFE for plain HTML.

terminal 
# npm
$ npm install openseadragon

# or grab the CDN build
<script src="https://cdn.jsdelivr.net/npm/openseadragon@6/build/openseadragon/openseadragon.min.js"></script>

2. Add a sized container

The container div must have a width and a height before initialization. This is the single most common gotcha — an empty div renders nothing.

index.html 
<div
  id="viewer"
  style="width:100%; height:600px"
></div>
Heads up If you're inside a flex parent, set the container's height explicitly or give the parent a defined height. 100% with no anchor will collapse to zero.

3. Mount the viewer

Pass a tileSources URL — DZI, IIIF info.json, or a custom source. Everything else has sensible defaults; tweak as you go.

main.js esm · 18 kB 
import OpenSeadragon from "openseadragon";

const viewer = OpenSeadragon({
  id: "viewer",
  tileSources: "/tiles/painting.dzi",
  prefixUrl: "/openseadragon/images/",
  showNavigator: true,
  zoomPerScroll: 1.4,
  gestureSettingsMouse: { clickToZoom: false }
});

// listen for events whenever you're ready
viewer.addHandler("zoom", (e) => {
  console.log(viewer.viewport.getZoom());
});

4. Next steps

  • Read the Viewport & coordinates guide — it unlocks overlays.
  • Try multi-image worlds if you have more than one image.
  • Browse the examples gallery for working starting points.
  • If your tile pyramid doesn't exist yet, see Creating tiles.

Viewer · selected API

The most-reached-for methods on the Viewer instance. Full reference under API → Viewer.

NameReturnsSummary
.open(source)voidLoad a tile source (DZI, IIIF, custom). Re-callable.
.viewport.zoomTo(z, point?)ViewportAnimate zoom to z. Optional pivot point in image coords.
.viewport.panTo(point)ViewportAnimate pan to a normalized viewport point.
.addOverlay(el, location)voidAnchor a DOM element in image space. Tracks pan/zoom/rotation.
.world.addItem(tiledImage)voidAppend another image to the world. Use for multi-image.
.setFullScreen(bool)voidEnter or exit fullscreen mode.
.addHandler(event, fn)voidSubscribe to a viewer event. See Events.
.destroy()voidTear down listeners and DOM. Call before re-mounting.

Common events

EventPayloadFires when…
open{ source }A tile source has loaded and is ready.
zoom{ zoom }Viewport zoom changes. Throttle for UI.
pan{ center }Viewport center changes.
tile-loaded{ tile }A tile image has decoded and entered the world.
canvas-click{ position }Pointer event in canvas coordinates.
animation{ }Per-frame while the viewport is animating.

Migrating from v5

Most v4 code runs unchanged on v5. The breaking surface is small and scoped. The full table is in the v5 → v6 guide.

  • The default drawer is still canvas. Opt into webgl via drawer: "webgl".
  • useCanvas was removed (it's been the default for a decade).
  • Plugin lifecycle now expects an explicit destroy(); old plugins may leak listeners on hot-reload.
  • IIIF 3.0 image profiles are recognized without the legacy shim.
Upgrade in one minute npm install openseadragon@latest — your existing viewer config is almost certainly fine. Read the migration page only if you wrote a custom drawer or a plugin that touches internal tile state.
